# 技能分类与推断字典 (Skill Taxonomy)

用于：(1) 将用户技能拆分为能力→子能力层级；(2) 从已知技能推断用户可能具备但未声明的相关能力；(3) 为简历技能板块生成更具体的应用场景描述。

AI 使用规则：
- **拆分**：用户说"熟悉 Claude Code"→ 拆为 AI能力 > LLM工具 > Claude Code
- **推断**：用户有 Claude Code + SQL → 可能具备"AI辅助数据分析"能力 → 询问用户确认，不自动写入
- **描述**：技能板块从"会用Claude Code"升级为"掌握Claude Code，用于业务程序搭建与自动化工作流"
- **诚实**：推断的能力需要用户确认后才写入简历。推断时标注"AI推测，请确认"

## AI 能力

### LLM 工具
- Claude Code → 业务程序搭建、自动化工作流、数据分析与可视化、代码生成与调试
- ChatGPT / GPT-4 → 文本生成、内容策略、翻译润色、创意策划
- Coze / Dify → Agent搭建、对话流设计、RAG知识库构建、多工具编排
- 文心一言 / 通义千问 → 中文内容生成、行业报告撰写

### Prompt 工程
- few-shot 示例设计 → 输出格式控制、语气约束、多场景适配
- Persona 控制 → 角色扮演、敌意评估、专业领域模拟
- 输出约束规则 → JSON格式化、字数控制、模板填充
- 迭代评估 → A/B测试Prompt效果、人工评分、自动指标验证

### LLM 评估
- 自动指标 → BLEU、chrF++、BERTScore、COMET
- 人工评估体系 → 评分矩阵设计、Kappa一致性、标注SOP
- 多模型对比 → API调用、平行语料库构建、显著性检验

### AI 应用落地
- 模型微调 / RAG → 知识库搭建、向量检索、上下文增强
- AI Native产品设计 → 对话交互、流式输出、fallback策略
- 内容审核AI → NSFW检测、合规拦截、柔性防御

## 数据分析

### SQL
- 数据提取 → SELECT / JOIN / 子查询、多表关联
- 归因分析 → 漏斗分析、异常定位、流失归因
- 用户分群 → RFM模型、行为聚类、生命周期分层
- 看板搭建 → 指标字典、自动化报表、数据可视化

### Python
- 数据处理 → pandas、numpy、数据清洗、格式转换
- 自动化脚本 → 文件批处理、定时任务、API数据拉取
- 统计检验 → t检验、卡方检验、Friedman检验、Wilcoxon事后检验
- 可视化 → matplotlib、seaborn、数据看板

### Excel
- 数据透视表 → 多维交叉分析、动态报表
- 可视化 → 图表制作、条件格式、动态看板
- 公式函数 → VLOOKUP、INDEX-MATCH、数组公式

## 推断规则

当用户在简历或素材库中声明了某个子技能时，AI 可按以下规则推断相关能力：

| 已知技能 | 可推断（需用户确认） | 推断依据 |
|---------|---------|---------|
| Claude Code | 自动化脚本编写、基础数据分析 | Claude Code 常用于这些场景 |
| SQL + Python | 数据清洗Pipeline、AB测试分析 | 两者结合通常用于数据工程 |
| Prompt工程 (few-shot) | Persona控制、输出约束设计 | 同一个能力链上的进阶技能 |
| PRD撰写 | 用户访谈、竞品分析、原型设计 | PRD 产出通常需要前期调研 |
| Python API调用 | JSON处理、多模型集成 | API 调用的标准配套能力 |
| CAT工具 (Trados) | 术语管理、翻译质量评估 | 翻译技术栈的标准组件 |

**推断格式**：
```
基于你已有的 [已知技能]，推断你可能也具备 [推断能力]。
例如：将 Claude Code 用于 [具体应用场景描述]。
是否需要将这条加入你的技能列表？（确认 / 跳过 / 修改描述）
```

**禁止**：直接写入未经用户确认的推断能力。推断必须附带场景描述，不允许只写技能名。
